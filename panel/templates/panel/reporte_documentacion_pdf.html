Arquitectura, Flujos de Datos y Especificaciones de Clase

1. Filosofía del Proyecto y Objetivos Técnicos

NetScanner es un sistema de auditoría de red de nivel profesional diseñado bajo el paradigma de Seguridad Pasiva. El objetivo primordial es obtener visibilidad completa de la infraestructura de red sin interactuar directamente con los activos mediante escaneos de puertos intrusivos o inyecciones de paquetes que puedan desestabilizar sistemas industriales (OT) o activar sistemas de prevención de intrusiones (IPS).

Principios de Diseño:

  • Invisibilidad Operativa: No genera tráfico detectable por firewalls o IDS corporativos.
  • Arquitectura desacoplada: Uso estricto del patrón Service-Repository para facilitar el mantenimiento.
  • Persistencia NoSQL: MongoDB Atlas para manejar esquemas de dispositivos altamente variables.

4. Núcleo de Gestión (Capa Managers)

Esta capa interactúa directamente con los recursos del sistema y controladores externos.

network_manager.py

Descripción Técnica:

Implementa la abstracción de red utilizando subprocesos del sistema y la librería psutil. Su función es actuar como un "sniffer" de configuración y estado.

Métodos Críticos:

  • obtener_conexiones_observadas(): Utiliza llamadas al kernel mediante psutil.net_connections para identificar sockets abiertos y sus destinos remotos.
  • leer_tabla_arp(): Ejecuta comandos nativos del SO para extraer la caché de resolución de direcciones física-lógica.
  • obtener_hostname(): Implementa una resolución DNS inversa local recursiva.
mongo_manager.py

Descripción Técnica:

Gestiona el ciclo de vida de la conexión con MongoDB. Implementa el patrón Singleton para evitar el agotamiento de sockets en el servidor de base de datos.

  • Manejo dinámico de URIs de conexión.
  • Validación de estado mediante comandos de ping administrativos a la instancia de Atlas.

3. Lógica de Persistencia (Capa Repositories)

historial_repository.py

Especificación:

Gestiona la serie temporal de auditoría. Permite la reconstrucción histórica de la red.

  • listar_todos_limpio(): Implementa la lógica de sanitización de documentos BSON para su consumo en interfaces JSON/Web.
  • buscar_primer_registro_por_mac(): Vital para el análisis forense y detección de intrusiones (primera vez que un dispositivo fue visto).
activos_repository.py

Especificación:

Mantiene el estado "Hot" de la red. Esta colección se sincroniza en cada ciclo de observación para alimentar la vista de Topología.

  • contar_por_tipo() / contar_por_fabricante(): Utiliza el motor de agregación de MongoDB para generar estadísticas en tiempo real con alta eficiencia (O(n)).

4. Orquestación y Negocio (Capa Services)

scanner_service.py

Descripción de Proceso:

Es el director de orquesta. Implementa el algoritmo de descubrimiento pasivo en 4 fases:

  1. Extracción de topología lógica (ARP).
  2. Correlación de sockets activos (psutil).
  3. Enriquecimiento de datos (Vendor & Classification).
  4. Evaluación de criticidad (Risk Analysis).
device_classifier_service.py

Motor de Clasificación:

Implementa un motor de inferencia basado en pesos. Analiza patrones en hostnames (ej: "SRV-" para servidores), fabricantes (ej: "Cisco" para infraestructura) y servicios observados para asignar una categoría al dispositivo.

risk_service.py

Algoritmo de Riesgo:

Calcula un vector de amenaza basado en la exposición de servicios conocidos, vulnerabilidades teóricas del fabricante y el rol del dispositivo en la red.
